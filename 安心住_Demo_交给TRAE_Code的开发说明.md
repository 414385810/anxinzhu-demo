# 安心住 Demo 开发说明

这份说明用于交给 TRAE Code，目标是从零重做一个更有完成度的手机端 H5 Demo。请不要沿用现有粗糙 Demo 的界面结构，只保留“拍照上传、AI 分析、风险识别、预算方案、双份报告”这个体验方向。

## 项目定位

项目名称：安心住

一句话定位：面向长辈居家空间安全隐患的 AI 自查工具，初赛阶段先聚焦老人夜间起夜时“床到卫生间”这条动线，帮助家庭发现风险，并给出低打扰、低成本的优化建议。

核心价值：老人居家安全，防患胜于救险。

参赛方向：社会服务 + 社会公益：智慧助老。

产品不是“老人摔倒后通知家人”的报警工具，也不是全屋适老化大改造方案。它要先帮普通家庭看见一个具体场景里的风险：爸妈夜里从床边起身、经过地毯和家具边角、走到卫生间入口，这几米路上有没有容易摔倒、绊倒、磕碰或摸黑行走的问题。

## 作者背景

作者有产品造型设计背景，做过设计软件培训，现在从事家具行业和新媒体工作。平时接触家具、空间动线、用户表达和内容传播比较多，所以这个选题不是纯技术视角，而是从真实家庭空间和普通家庭沟通问题出发。

Demo 里可以适当体现这三个标签：

- 产品造型设计背景
- 家具行业一线观察
- 熟悉空间动线与用户沟通

## Demo 目标

需要做一个看起来像真实产品原型的手机端 H5，不是普通网页说明页。

评委打开后应该能在 1 分钟内体验到完整流程：

1. 选择或上传一张“床到卫生间动线”的照片
2. 看到 AI 正在分析空间风险的过程
3. 看到照片上出现风险点标记
4. 查看每个风险点的解释和优先级
5. 选择预算范围
6. 生成两份报告：子女版和老人版

这是初赛 Demo，可以采用“模拟 AI 分析”的方式，不需要真实接入视觉模型 API。但体验上要像真实 AI 产品，不能像静态说明页。

## 推荐技术方案

优先使用：

- Vite + React + TypeScript
- CSS 或 Tailwind 均可
- 不需要后端
- 不需要真实 AI API
- 所有数据本地模拟
- 最终可打包成静态网页

如果为了快速交付，也可以做成单文件 HTML，但界面完成度要高。建议使用 React，因为多步流程、状态切换、报告生成、图片预览会更清晰。

推荐输出目录：

`/workspace/anxinzhu-demo`

推荐启动方式：

```bash
npm install
npm run dev
```

最终需要提供：

- 可运行的开发版本
- 可打包的静态版本
- 简短 README，说明如何启动、如何体验

## 页面结构

### 首页

目的：让评委立刻知道这是什么。

内容建议：

- 品牌名：安心住
- 主标题：帮爸妈看见夜里那几米路
- 副标题：AI 居家安全自查 Demo，先从“床到卫生间”开始
- 核心标签：拍照自查 / AI 识别风险 / 预算方案 / 双份报告
- 主按钮：开始自查

首页要有一点产品感，可以加入一条简化的夜间动线插画：床、地面、家具边角、卫生间入口、柔和灯光。

### 上传照片页

目的：让用户感觉自己正在输入真实家庭空间。

功能要求：

- 支持上传图片
- 支持手机拍照入口
- 上传后显示图片预览
- 可以重新上传
- 如果用户不上传，也提供一个“使用示例空间体验”的按钮，避免评委现场没有合适照片时流程卡住

页面文案：

> 请拍下或上传爸妈夜间从床到卫生间常走的那条路。尽量包含床边、地面、家具边角和卫生间入口。

### AI 分析页

目的：让 Demo 有“AI 正在工作”的可信体验。

功能要求：

- 显示用户上传的照片，或示例空间图
- 图片上出现扫描线
- 出现 4-5 个风险点定位标记
- 每个标记可以有不同颜色：高风险、中风险、低风险
- 分析过程中显示阶段性文案

分析文案建议：

- 正在识别床到卫生间的动线路径
- 正在检测地面障碍和地毯风险
- 正在分析夜间照明覆盖情况
- 正在评估家具边角碰撞风险
- 正在检查卫生间入口湿滑和门槛风险
- 正在生成风险优先级

分析时长建议 3-5 秒。不要太长，评委体验要流畅。

### 风险结果页

目的：体现核心能力。

这一页要重点做漂亮，不能只是列表。建议做成“照片 + 风险标记 + 下方风险卡片”的组合。

风险点建议：

1. 床边起身支撑不足
   - 风险等级：高
   - 说明：老人半睡半醒起身时，如果没有稳定扶手或支撑物，容易站不稳。
   - 建议：床边增加稳定支撑，或调整床头柜位置形成可扶握点。

2. 地面有地毯、拖鞋或杂物
   - 风险等级：高
   - 说明：夜间视线差，地毯边缘、拖鞋和杂物最容易造成绊倒。
   - 建议：固定地毯边缘，清理动线上的杂物，把拖鞋放到固定位置。

3. 夜间照明不足
   - 风险等级：中
   - 说明：摸黑行走会增加绊倒和碰撞概率，突然开强光也容易刺眼。
   - 建议：增加柔和感应夜灯，优先布置床边、转角和卫生间入口。

4. 家具边角靠近动线
   - 风险等级：中
   - 说明：茶几、柜角、床角靠近路线时，夜间更容易磕碰。
   - 建议：微调家具位置，或增加防撞角。

5. 卫生间入口湿滑或有门槛
   - 风险等级：中 / 低
   - 说明：卫生间入口容易湿滑，门槛也可能成为绊脚点。
   - 建议：增加防滑垫、防滑条，保持入口干燥。

需要展示一个综合风险结论，比如：

> 本次识别到 5 个风险点，其中 2 个高风险、2 个中风险、1 个低风险。建议先处理地面障碍和夜间照明。

### 预算选择页

目的：体现安心住的差异化，不是给用户一个昂贵的大改造方案，而是在预算范围内优先处理最重要的问题。

预算选项：

1. 低打扰基础版
   - 适合预算有限、父母抗拒改动的家庭
   - 重点：移开杂物、固定地毯、补充夜灯

2. 安全优先版
   - 适合老人腿脚不便、夜间起夜频繁的家庭
   - 重点：床边支撑、防滑、防撞、照明

3. 舒适增强版
   - 适合愿意做更多改善的家庭
   - 重点：连续夜灯、家具微调、常用物品下移、卫生间入口优化

不要写“保证最终花费接近 AI 估算”。稳妥表达：

> 方案会在预算范围内优先排序风险点，实际价格会因地区、材料和施工方式不同而变化。

### 报告页

目的：体现“家庭沟通”差异化。

报告页需要有两个 Tab：

- 子女版报告
- 老人版报告

子女版口吻更理性，强调风险、优先级和低成本处理。

老人版口吻更温和，避免让老人觉得“被否定”或“家里不好”。它应该像一段可以直接给爸妈看的解释。

示例：

子女版：

> 当前优先处理地面障碍和夜间照明。建议先清理床到卫生间路线上的杂物，固定或撤掉松动地毯，并增加柔和感应夜灯。这些调整成本低、打扰小，但能降低夜间绊倒风险。

老人版：

> 不是要大改家里，也不是说家里不好。只是晚上起夜时光线比较暗，路上有些小东西容易绊脚。我们先把最常走的这条路理顺一点，让晚上起来更省力、更安心。

报告页可以有“保存报告”按钮，但不要求真的生成 PDF。点击后可以弹出 Toast：

> 报告已生成，可截图保存或分享给家人。

## 内容边界

初赛阶段只聚焦：

不要加入太多空间：

- 厨房
- 客厅
- 阳台
- 楼梯
- 全屋户型

这些可以作为复赛扩展方向，但初赛 Demo 不要展开。

## 需要避免的问题

- 不要做成普通宣传落地页
- 不要只做静态卡片
- 不要使用过多 emoji
